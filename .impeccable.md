# Impeccable Design Context — WAIC Side Events

## Design Context

### Users
- **参会者**：中国 AI 从业者/爱好者，绝大多数从微信分享链接在手机上打开（375px 一等公民），见惯小红书/即刻/lu.ma 的精致度，30 秒决定报不报名。
- **主办方**：社区组织者，判断"这个平台配不配放我的活动"——平台档次感就是背书。
- **现场工作人员**：志愿者，手机单手操作核销台，强反馈优先。

### Brand Personality
三个词：**编辑感、笃定、利落**。情绪目标：可信赖的城市活动指南（像一本做得很好的会展特刊），不是模板站。本项目是 WaytoAGI 2.0 的先导项目，视觉要"更高级"。

### Aesthetic Direction
- **视觉语言**：编辑感实用主义（editorial-utility）。暖纸白 `#fafaf8` + 墨色 `#17181c` + 钴蓝 `#2745e8` 单一强调色；amber `#f59e0b` 只做「进行中」状态语义。
- **字体**：Geist Variable（拉丁）+ PingFang SC 系统栈（中文），标题重字重+tight tracking 做编辑感，不引 CJK webfont。
- **形状体系（锁定）**：卡片 rounded-2xl(16px)、按钮/chip 全圆角 pill、输入框 rounded-lg(10px)。
- **深度体系**：卡片默认态即有轻投影（tinted shadow，非纯黑），hover 抬升；禁止"全平+只有 hover 才显形"。
- **反面清单（Kane 明确反对）**：AI 紫/渐变 slop、wix 模板感、Inter+slate 默认脸、灰块图片兜底、纯文字表格式页面、素到像没设计过。
- 深浅双模式都要成立。

### Design Principles
1. **内容即设计**：海报和活动信息是主角，装饰服务于扫读速度。
2. **默认态就完整**：层次、阴影、兜底视觉在静止状态就成立，不依赖 hover。
3. **单一强调色纪律**：钴蓝管动作与身份，别处不抢戏；品牌色要有"存在面积"（母题图形），不只出现在小字上。
4. **移动优先的触达感**：拇指可达、强反馈、safe-area。
5. **每屏一个焦点**：信息分级明确，日期/时间永远最先被看到。
